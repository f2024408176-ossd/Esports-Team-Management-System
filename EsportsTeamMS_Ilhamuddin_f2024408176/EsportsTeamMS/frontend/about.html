About EsportsMS

A full-stack web application developed for the University of Management and Technology OSSD course final project.

Problem Statement

Why EsportsMS?

The esports industry is growing rapidly, and professional as well as amateur gaming teams require an efficient system to manage players, tournaments, team schedules, match results, and training sessions.

Most small esports organizations rely on spreadsheets or manual records, which leads to poor organization, data loss, and inefficient team management.

EsportsMS provides a centralized platform where team managers can manage everything in one place, backed by a robust FastAPI backend and PostgreSQL database hosted on Supabase.

🎯

Project Objective

Build a complete, deployed full-stack web application that demonstrates CRUD operations, database integration, JWT auth, and API development using FastAPI + PostgreSQL.

👥

Target Users

Esports Team Managers, Coaches, Team Owners, Professional Players, Amateur Gaming Teams, and Tournament Coordinators.

Features

Main Features

🔐

User Authentication

JWT-based registration, secure login, and role-based access control for different user types.

🎮

Team Management

Create teams, assign coaches, manage team information and view all team details in one place.

👤

Player Management

Add players, update gamer profiles, assign roles (IGL/Support/Fragger), and remove players.

🏆

Tournament Management

Add tournaments, set locations, prize pools, dates and track team's tournament participation.

⚔️

Match Management

Schedule matches, record results (Win/Loss/Draw), view full match history with scores.

📅

Training Sessions

Create training sessions, manage practice schedules, set focus areas and track durations.

📊

Analytics Dashboard

Visual team win rates, match statistics, tournament performance and player metrics.

🔍

Search & Filter

Search players by name/tag, filter matches by result, and filter tournaments by date.
